이벤트(Event)

    * Event 뜻 : 행동, 동작, 행위, 행사 ==> 브라우저에서 할 수 있는 동작
    ex) click, keydown, keypress, keyup, mouseover(hover), mouseon, mouseout, mouseleave,
        change, drag, submit, focus, blur, ...
        
HTML 이벤트 종류 참고
    
    * 이벤트 리스너 (Event Listener)  // Listener : 감지기
      - 브라우저에서 발생하는 이벤트를 감지
      - 이벤트가 감지될 때 까지 대기
      - 이벤트가 감지 되었을 때 연결된 기능(함수)를 수행
      - 이벤트 리스너 작성법
        1) on이벤트명
            ex) onclick, onchange, onsubmit, onkeydown, ...

        2) 요소.addEventListener("이벤트명", 함수);

    * 이벤트 핸들러 (Event Handler)   // Handler : 처리기
      - 이벤트 리스너에 연결된 기능(함수)
      - 이벤트가 감지 되었을 때 수행하거나 처리하는 내용을 작성한 함수

인라인 이벤트 모델(Inline)

    HTML 요소에 직접 이벤트 리스너 = "이벤트 핸들러" 형식으로 작성하는 방법 (지금까지 했던 방식)
    * 최근 많이 사용 X , 종종 사용 O (지양)

    * 인라인 이벤트 모델 작성 시 이벤트 핸들러 () 안에 this를 작성하면 이벤트가 발생한 요소를 나타낸다!
      ex) onclick = testFn(this)  / this == 클릭된 요소

고전 이벤트 모델

    - HTML 요소 이벤트 관련 코드를 직접 작성하지 않고 JS에서 DOM을 이용해 원하는 이벤트 리스너 / 핸들러를 직접 작성하는 방법
    
    *단점 : 하나의 요소에 동일한 이벤트 리스터/핸들러 작성 불가능(마지막 작성한 이벤트 리스터/핸들러 코드가 적용)(덮어쓰기)

표준 이벤트 모델( 요소.addEventListener() )

    - W3C (웹 표준 재정 단체) 에서 공식적으로 지정한 이벤트 모델 (이게 표준이야!)
    - 고전 이벤트 모델의 단점인 한 요소에 같은 이벤트 리스너 / 핸들러 작성 1개 만 되는 것을 해결!
